The **Foundation Nova CLI** lets you generate files, apply recipes, scaffold projects, and run utilities.

:::tip Step by step
Not sure what to type? Build the command **one section at a time**.

Start with `nova`, then the **command** and **subcommand** (if available), and finally the **options** like `--all` or `-a`. Each step shows **inline guidance** so you don't wonder what to do next.
:::

## Get Started

<Tabs groupId="install-state">
  <TabItem value="nova" label="nova (installed)" default>

    ```bash
    nova <command> <subcommand> [option]
    ```

  </TabItem>
  <TabItem value="npx" label="npx (no install)">

    ```bash
    npx --yes @cbnventures/foundation-nova@latest <command> <subcommand> [option]
    ```

  </TabItem>
</Tabs>

- **Options are recommended**. Always type the option (**flags / switches**) you want, like `--all` or `-a`. No assumptions. This helps you learn the command faster.
- **Build it step by step**. Start with `nova`. Add the **command**. Add the **subcommand** if needed. Finish with **options** like `--all` or `-a`. Repeat until it feels natural.

## Commands

All commands below **show a help screen when run with the `--help` or `-h` option**, so you can explore safely and avoid accidental runs.

### Generators

Create **semi-tailored** versions of supported vendor and project essentials, populated with values from `nova.config.json` at the monorepo root.

:::info
All commands in "Generators" are shaped as `nova generate <subcommand>` or `nova gen <subcommand>`.
:::

| Subcommand                                                                | What It Does                                                                                                                 |
|---------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------|
| [`aws-amplify`](/docs/cli/generators/aws/amplify)                         | Create a `amplify.yml` file for [AWS Amplify](https://aws.amazon.com/amplify/) deployments                                   |
| [`cloudflare-workers`](/docs/cli/generators/cloudflare/workers)           | Create a `wrangler.toml` file for [Cloudflare Workers](https://workers.cloudflare.com/) deployments                          |
| [`docker-compose`](/docs/cli/generators/docker/compose)                   | Create a `docker-compose.yml` file for [Docker](https://docs.docker.com/compose/) builds                                     |
| [`docker-file`](/docs/cli/generators/docker/file)                         | Create a `Dockerfile` for [Docker](https://docs.docker.com/reference/dockerfile/) builds                                     |
| [`docusaurus-config`](/docs/cli/generators/docusaurus/config)             | Create a `docusaurus.config.ts` file for [Docusaurus](https://docusaurus.io/) configuration                                  |
| [`github-funding`](/docs/cli/generators/github/funding)                   | Create a `./github/FUNDING.yml` file for [GitHub](https://github.com/) to enable sponsor links                               |
| [`github-issue-template`](/docs/cli/generators/github/issue-template)     | Create a `./github/ISSUE_TEMPLATE` directory for [GitHub](https://github.com/) to create issue templates                     |
| [`github-workflows`](/docs/cli/generators/github/workflows)               | Create a `./github/workflows` directory for [GitHub](https://github.com/) to enable CI/CD automation                         |
| [`must-haves-dotenv`](/docs/cli/generators/must-haves/dotenv)             | Create a `.env` file for managing local environment secrets across your project                                              |
| [`must-haves-editorconfig`](/docs/cli/generators/must-haves/editorconfig) | Create a `.editorconfig` file for managing consistent coding styles                                                          |
| [`must-haves-gitignore`](/docs/cli/generators/must-haves/gitignore)       | Create a `.gitignore` file for managing files that should be excluded from Git commits                                       |
| [`must-haves-license`](/docs/cli/generators/must-haves/license)           | Create a `LICENSE` file for managing project license agreements                                                              |
| [`must-haves-post-install`](/docs/cli/generators/must-haves/post-install) | Create a `post-install.ts` file for expanding on post-install controls                                                       |
| [`must-haves-read-me`](/docs/cli/generators/must-haves/read-me)           | Create a baseline `README.md` file for for your project                                                                      |
| [`nextjs-config`](/docs/cli/generators/nextjs/config)                     | Create a `next.config.mjs` file for [Next.js](https://nextjs.org/docs/app/api-reference/config/next-config-js) configuration |
| [`vite-config`](/docs/cli/generators/vite/config)                         | Create a `vite.config.mjs` file for [Vite](https://vite.dev/config/) configuration                                           |

### Recipes

Apply **scripted edits** that automate routine maintenance, using values from `nova.config.json` at the monorepo root.

:::info
All commands in "Recipes" are shaped as `nova recipe <subcommand>` or `nova rcp <subcommand>`.
:::

| Subcommand                                       | What It Does                                                                |
|--------------------------------------------------|-----------------------------------------------------------------------------|
| [`freeze-deps`](/docs/cli/recipes/freeze-deps)   | Freezes `package.json` dependencies to prevent unwanted version changes     |
| [`sync-pkg-mgr`](/docs/cli/recipes/sync-pkg-mgr) | Syncs the preferred package manager to be used across the entire repository |

### Scaffolding

Bootstrap **full project starters** for popular builders and tools. All scaffolds are built as a monorepo, and includes Docusaurus built-in.

:::info
All commands in "Scaffolding" are shaped as `nova scaffold <subcommand>` or `nova scaf <subcommand>`.
:::

| Subcommand                               | Project Type |
|------------------------------------------|--------------|
| [`nextjs`](/docs/cli/scaffolding/nextjs) | Next.js      |

### Utilities

Tools for **diagnostics**, **quick checks**, and **development helpers**.

:::info
All commands in "Utilities" are shaped as `nova utility <subcommand>` or `nova util <subcommand>`.
:::

| Subcommand                                     | Shorthand | What It Does                                                                                                 |
|------------------------------------------------|-----------|--------------------------------------------------------------------------------------------------------------|
| [`initialize`](/docs/cli/utilities/initialize) | `init`    | Generate a new `nova.config.json` configuration file for use with `nova generate` and `nova recipe` commands |
| [`inspect`](/docs/cli/utilities/inspect)       | `ins`     | Prints the Foundation Nova, ESLint, or TypeScript configuration                                              |
| [`version`](/docs/cli/utilities/version)       | `ver`     | Generate a Markdown-ready snapshot of your development stack versions (e.g., `node --version`)               |

## Troubleshooting

### Exit Codes

| Code | Meaning | Result                           |
|------|---------|----------------------------------|
| `0`  | Success | Response of the executed command |
| `1`  | Error   | Help text displayed              |

### Quick Fixes
- Prefer **explicit command, subcommands, and options** for clarity, fewer mistakes, and predictable behavior. For example, `nova utility version --all` instead of the alternate variants.
